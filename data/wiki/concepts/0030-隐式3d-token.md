# 隐式3D token

## 定义

隐式 3D token 是一种在视觉语言模型中承载几何先验的中间表示。它不直接依赖深度相机、显式 3D 地图或人工构建的场景模型，而是从单目视频的多帧变化中提取出关于场景结构与相机运动的隐含三维线索，用来增强后续的语言推理能力 [[paper:3]]。

在 VLM-3R 中，隐式 3D token 主要分为两类：一类是描述场景空间结构的 spatial tokens，另一类是描述相机运动和视角变化的 view tokens。前者帮助模型理解物体、区域和空间布局之间的关系，后者帮助模型判断拍摄者或相机如何移动、视角如何变化，以及这些变化如何影响物体相对位置 [[paper:3]]。

## 作用机制

隐式 3D token 的核心作用，是把普通视频中的视觉语义表示补上一层“空间感”。VLM-3R 一方面用视觉编码器提取图像中的物体类别、外观和局部细节，另一方面用几何编码器从多帧对应关系中恢复稠密点图、相机内外参与相对位姿，并进一步抽取 spatial tokens 和 view tokens [[paper:3]]。

这些几何 token 随后通过 Spatial-Visual-View Fusion 模块注入视觉 token。融合后的表示既保留“画面里有什么”的语义信息，也包含“它们在三维空间中如何分布”以及“相机如何运动”的几何信息。最后，这些表示被投影到语言模型空间，与用户问题一起用于回答距离、方向、大小、路线和时间变化等问题 [[paper:3]]。

## 意义

隐式 3D token 的价值在于，它为 VLM 提供了一种不依赖显式 3D 输入的几何增强路径。根据 VLM-3R 的实验，仅使用单目视频，模型也能在空间理解和时空推理任务上获得明显提升，并接近部分使用真值深度信息的方法 [[paper:3]]。

这说明隐式 3D token 可以被看作一种“压缩后的脑内三维沙盘”：它不一定显式输出完整的 3D 场景，但能把几何结构、视角变化和相机运动编码进模型可用的 token 表示中，使语言模型在回答空间问题时不只依赖二维外观和语义线索 [[paper:3]]。

## 局限与未解问题

现有材料主要来自 VLM-3R，因此对隐式 3D token 的讨论仍集中在单一框架内。尚不清楚这类 token 在不同几何编码器、不同底座 VLM、不同视频域中的泛化边界如何，也不清楚 spatial tokens 与 view tokens 各自对不同类型空间推理任务的贡献是否稳定 [[paper:3]]。

此外，材料显示大规模 3D 重建式指令数据对效果很关键，但尚未充分说明隐式 3D token 本身与训练数据规模、任务设计之间的相对贡献。换言之，性能提升究竟主要来自更好的几何表示，还是来自更系统的空间指令数据，仍需要进一步拆解 [[paper:3]]。
