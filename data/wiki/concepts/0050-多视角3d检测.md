# 多视角3D检测

## 定义

多视角3D检测在这里指从车身周围多个相机视角中恢复场景的三维结构，并产生带有空间几何先验的目标表示。它不只是判断图像里“有什么”，而是进一步给出实例级目标在3D空间中的位置与候选框，因此适合作为自动驾驶视觉指代任务中的定位基础。

在 NuGrounding 中，多视角3D检测由 BEV-based Detector 承担：模型先把六路相机图像编码到鸟瞰视角特征，再通过查询式检测解码器生成 object query。这些 object query 同时包含目标语义和3D几何信息，随后被送入多模态大模型与语言指令对齐 [[paper:12]]。

## 在框架中的作用

NuGrounding 将多视角3D检测视为“几何测绘员”：它负责从多相机画面中整理出场景里的候选目标及其空间位置，而多模态大模型负责理解自然语言描述。最终，Fusion Decoder 将语言语义与3D空间信息融合，用于找出被指代的具体目标 [[paper:12]]。

这种设计的关键价值在于弥补多模态大模型空间定位能力不足的问题。单靠语言和图像理解，模型可能知道描述的是“前右方黑色轿车”，但难以稳定地落到精确3D实例；引入专业3D检测器后，模型可以在已有候选物体和几何先验上进行选择与对齐 [[paper:12]]。

## 经验结论

论文的消融结果表明，object query 比直接使用 BEV feature 或 blind 输入更适合作为视觉输入，因为它天然携带实例级3D几何先验，更容易和语言描述对齐 [[paper:12]]。加入 context query 后，mAP 和 NDS 也进一步提升，说明语言语义与检测器输出之间的跨模态聚合是有效的 [[paper:12]]。

整体上，多视角3D检测在该工作中不是最终任务本身，而是多模态3D grounding 的几何基础设施：它提供候选目标、实例边界和空间结构，让语言模型能够在更可靠的3D表示上完成指代定位 [[paper:12]]。
