# DriveLM: Driving with Graph Visual Question Answering

## 一句话定位

[[DriveLM]] 将端到端自动驾驶建模为一个 [[Graph VQA]] 流程：先回答“看见什么、会怎么动、我该怎么开”这一系列有依赖关系的图结构问题，再生成驾驶行为描述与未来轨迹，从而把 [[视觉语言模型]] 引入可解释驾驶决策。

## 核心贡献

- 提出 [[Graph VQA]] 作为驾驶代理任务，把驾驶理解组织成有向无环图中的问答节点，显式建模：
  - 多个交通参与者之间的关系
  - 感知、预测、规划、行为、运动各阶段之间的依赖
- 构建了两套数据：
  - [[DriveLM-nuScenes]]
  - [[DriveLM-CARLA]]
- 提出 [[DriveLM-Agent]] 基线，结合：
  - 通用 [[视觉语言模型]]（文中基于 [[BLIP-2]]）
  - [[图提示]]
  - [[LoRA]] 参数高效微调
  - [[轨迹离散化]]
- 将语言中间表示引入驾驶闭环中：先输出行为描述，再生成轨迹，提升了解释性与零样本泛化能力。

## 方法

### 整体思路

论文的核心思想是把自动驾驶从“一步直接出轨迹”改成“先推理再行动”。

输入是一帧场景图像。模型按图结构问答逐步完成：

1. **感知阶段**：识别关键物体及其位置、状态  
2. **预测阶段**：根据前面答案推断其他交通参与者未来可能动作与交互  
3. **规划阶段**：判断自车可执行或危险的动作  
4. **行为阶段**：汇总前述信息，生成一句自然语言驾驶行为描述  
5. **运动阶段**：结合图像与行为描述，输出未来轨迹

其中，每个问题在提问时会把其父节点问答作为上下文拼接进提示词，因此信息沿图传播，这对应 [[图提示]] 的做法。

### [[Graph VQA]]

[[Graph VQA]] 是本文的核心任务建模方式。它并不是只对整幅场景做一个总问答，也不是围绕单一目标做简单链式追问，而是把驾驶场景中的多对象关系、多阶段依赖统一到图结构中。

相比已有驾驶问答形式，它的特点是：

- 同时考虑多个交通参与者
- 显式表达对象之间与任务阶段之间的逻辑依赖
- 适合作为完整驾驶推理代理任务

### 行为与轨迹生成

在高层推理完成后，模型先生成行为表示：

\[
B = (B_{sp}, B_{st})
\]

即把驾驶行为拆成速度类别和转向类别两部分，作为从高层推理到低层轨迹的桥梁。这一中间层对应 [[行为表征]]。

随后，未来轨迹不是直接回归，而是先改写为增量序列，再做离散化预测：

\[
\{x, y\}_{dist} = \{(\delta x_1, \delta y_1), \ldots, (\delta x_N, \delta y_N)\}
\]

其中：

\[
\delta x_i = x_i - x_{i-1}
\]

\[
\delta y_i = y_i - y_{i-1}
\]

这对应 [[轨迹离散化]] 或轨迹 token 化。论文中先预测离散 token，再还原为连续轨迹点。

### 与已有工作的区别

论文认为，过去把 [[视觉语言模型]] 用于驾驶主要有两类：

- 场景级问答：只给整体解释
- 面向单目标的链式问答：缺少多对象联合推理

而本文的改进在于：

- 用图而不是单链组织问题
- 用语言中间表示连接高层推理与低层控制
- 保持端到端训练框架，但推理过程更接近人类分步驾驶判断

## 实验与结论

### 主要结论

论文的实验结论是：显式的图结构推理和行为中间层，能明显改善开放环规划表现，并提升跨传感器配置的零样本泛化。

### 开放环规划结果

在 [[DriveLM-nuScenes]] 上，与直接做轨迹预测的基线相比，[[DriveLM-Agent]] 表现更强。

文中给出的结果包括：

- [[BLIP-RT-2]]：
  - ADE = 2.63
  - 碰撞率 = 2.77
- [[DriveLM-Agent]]（None 版本）：
  - ADE = 1.39
  - 碰撞率 = 1.67
- [[UniAD-Single]]：
  - ADE = 1.80
  - 碰撞率 = 2.62

这说明引入行为中间层后，轨迹质量和安全性指标都有明显改善。

### 零样本泛化

在只用 [[nuScenes]] 训练、到 [[Waymo]] 测试的设置下，图结构版本优于其他组织方式：

- Graph 版本：
  - 行为准确率 = 39.73%
  - 运动 ADE = 2.63
- None 版本：
  - 行为准确率 = 35.70%
  - 运动 ADE = 2.76
- Chain 版本：
  - 行为准确率 = 34.62%
  - 运动 ADE = 2.85
- [[UniAD-Single]]：
  - 运动 ADE = 4.16

论文据此认为，[[Graph VQA]] 带来的上下文组织方式对跨传感器配置泛化尤其有帮助。

### 论文的整体判断

这项工作更像是一种范式探索：把 [[视觉语言模型]] 真正嵌入自动驾驶推理流程，用“图结构问答 + 行为描述 + 轨迹输出”的方式，搭建出可解释的驾驶代理框架。

## 限制与待解

论文中提到的边界主要包括：

- 主要基于单帧图像推理，时序信息利用不如视频模型充分
- [[Graph VQA]] 的质量高度依赖标注与问题模板设计
- [[轨迹离散化]] 对精细连续控制的表达能力可能不如专门连续控制器
- [[DriveLM-CARLA]] 中大量标注来自规则系统，与真实世界仍存在分布差异
- 普通 [[视觉语言模型]] 在 [[GVQA]] 上的能力仍然有限，说明可靠图推理仍未解决
